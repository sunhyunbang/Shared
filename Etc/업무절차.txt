기획, 디자이너, 퍼블리셔 수정 요청 건
- HTML, CSS 및 텍스트 수정인 경우
  → 수정 요청자 JIRA 생성, 작업 요청(기획, 디자이너, 퍼블리셔) 
  → JIRA 번호로 git 브렌치 생성 후 작업(퍼블리셔)
  → 테스트 및 확인(기획, 퍼블리셔)
  → git pull request 요청(요청자 : 퍼블리셔, 승인자 : pull request 승인 담당자)
  → 배포 승인 요청(요청자 : 퍼블리셔, 승인자 : 배포 승인 담당자)
  → 배포(개발자, 배포자 : 배포 담당자)
  → 배포건 확인(담당자 모두)
   
- 개발 수정이 필요한 경우
  → 수정 요청자 JIRA 생성, 작업 요청(기획, 디자이너, 퍼블리셔) 
  → JIRA 번호로 git 브렌치 생성 후 작업(퍼블리셔)   
  → 퍼블 작업 본 테스트 및 확인(기획, 퍼블리셔)
  → 개발진행(개발자)
  → 개발 완료 본 테스트 및 확인(기획, 개발자)
  → git pull request 요청(요청자 : 개발자, 승인자 : pull request 승인 담당자)
  → 배포 승인 요청(요청자 : 개발자, 승인자 : 배포 승인 담당자)
  → 배포(개발자, 배포자 : 배포 담당자)
  → 배포건 확인(담당자 모두)
 
⇒ pull request 승인 담당자 : 이근우(구인)
   배포 승인 담당자 : 배정민
   
⇒ 배포일 :
  → 업무에 지장이 있는 배포 건
     매주 화, 목 오후 6시 ~ 8시
  → 업무에 지장이 없는 배포 건
     매주 화, 목 오전 10시 ~ 12시, 오후 2시 ~ 4시
  → 긴급 오류 건은 수정 후 즉시배포
