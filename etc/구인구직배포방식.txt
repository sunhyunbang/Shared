일반
1. develop Pull
2. feature/지라번호 브랜치 생성
3. 작업 완료 후 develop Pull 후 feature/지라번호 커밋&푸시
- 커밋 코멘트
type: subject 입력(예:fix: 회원아이디 유효성검사 오류처리)
- 커밋타입
feat : 새로운 기능 추가
fix : 버그 수정
doc : 문서
style : 코드 포멧팅, 세미콜론, 코드변경이 없는 경우
refactor : 코드 리펙토링
test : 테스트 코드, 리펙토링 테스트 코드
chore : 빌드 업무 수정, 패키지 매니저 수정

4. develop에 full request 요청. github 사이트로 이동 후 feature/지라번호 -> develop Pull requests 버튼 클릭 후 오른쪽 Assignees에 이근우 파트장님(cowto76) 추가
5. Merge pull request 이근우 파트장님이 처리
6. release/지라번호 브랜치 생성
7. release/지라번호 푸시
8. master와 머지. master 선택한 상태에서 release/지라번호 브랜치에 마우스 오른쪽 후 (현재 브랜치로 release/지라번호 병합) 선택
9. master 푸시
10. 완료

hotfixes
1. hotfixes/지라번호 생성
2. hotfixes/지라번호 푸시
3. master와 머지. master 선택한 상태에서 hotfixes/지라번호 브랜치에 마우스 오른쪽 후 (현재 브랜치로 hotfixes/지라번호 병합) 선택
4. master 푸시
5. develop과 머지. develop 선택한 상태에서 hotfixes/지라번호 브랜치에 마우스 오른쪽 후 (현재 브랜치로 hotfixes/지라번호 병합) 선택
6. develop 푸시
7. 완료
